# Mini-Bot 硬件技术方案

## 一、主控计算平台

### 推荐方案：Raspberry Pi 5 + ESP32-S3 双主控

| 平台 | 型号 | 适用场景 | 优劣势 |
|------|------|----------|--------|
| 高性能 | Raspberry Pi 5（4核Cortex-A76，2.4GHz） | 需运行完整Linux、ROS2、AI视觉 | 性能强，生态丰富，功耗约15W |
| 轻量级 | Raspberry Pi Zero 2W | 纯.NET IoT开发、简单交互 | 体积小、成本低 |
| 实时控制 | ESP32-S3（双核） | 电机底层控制、IMU读取 | 自带Wi-Fi/BT，支持AI指令加速 |

## 二、底盘与驱动

### 2.1 轮式方案对比

| 类型 | 推荐硬件 | 说明 |
|------|----------|------|
| 2轮差速 | Mini 2WD底盘套件 + 带编码器直流减速电机 | 结构简单，适合桌面移动 |
| 4轮麦克纳姆轮 | 4个编码器电机 + TB6612FNG / DRV8833驱动板 | 全向移动，灵活性高 |

### 2.2 驱动芯片

- **TB6612FNG**：双路PWM，控制2路直流电机
- **DRV8833**：双路H桥，控制2-4路直流电机

## 三、机械手（可选）

| 方案 | 自由度 | 特点 |
|------|--------|------|
| 入门DIY | 3-4 DOF，MG996R舵机 | 成本低，适合抓取轻物 |
| 专业开发 | 6 DOF，Dynamixel串行舵机 | 位置/力矩反馈，支持ROS，精度高 |
| 成熟套件 | ArmPi Pro | 树莓派+ROS+机械臂+麦克纳姆底盘，开箱即用 |

## 四、显示屏幕

| 尺寸 | 接口 | 推荐用途 |
|------|------|----------|
| 0.96" OLED | I²C | 小表情、电量、网络状态 |
| 2.4"-3.5" TFT/IPS | SPI（ILI9341） | 主显示界面，表情动画、状态信息 |
| 5"-7" HDMI | HDMI | 复杂UI、视频流显示 |

## 五、传感器选配

| 传感器 | 型号 | 用途 |
|--------|------|------|
| 姿态IMU | MPU6050 / ICM-42688P | 姿态检测、防倾倒 |
| 深度相机 | Intel RealSense D435i（可选） | 物体识别、SLAM、抓取定位 |
| 麦克风 | INMP441（I²S MEMS） | 语音交互 |
| 扬声器 | MAX98357A + 小喇叭 | 语音反馈 |

## 六、引脚分配（ESP32-S3）

```
GPIO 0-1:   UART0 (调试)
GPIO 43-44: UART1 (与Pi5通信)
GPIO 1-2:   I2C (IMU MPU6050)
GPIO 48:    SPI CLK (屏幕)
GPIO 45-47: SPI Data (屏幕)
GPIO 6-9:   PWM (4路舵机)
GPIO 10-13: 编码器输入
GPIO 14-15: 电机驱动PWM
GPIO 21-22: 电机驱动方向
```

## 三、核心 ROS2 节点设计

| 节点名称 | 功能 | Topic |
|---------|------|-------|
| `/chassis_driver` | 电机底层驱动，接收速度指令 | Sub: `/cmd_vel` |
| `/odometry` | 轮式里程计计算 | Pub: `/odom` |
| `/imu_driver` | IMU数据采集与滤波 | Pub: `/imu/data` |
| `/display_ui` | 屏幕表情/状态显示服务 | Service: `/set_face`, `/set_status` |
| `/arm_controller` | 机械臂轨迹控制（可选） | Action: `/follow_joint_trajectory` |
| `/voice_engine` | 语音唤醒与识别（可选） | Pub: `/voice_command` |

## 四、显示屏 UI 架构

```
┌─────────────────────────────────────────────────────────┐
│ 主循环（100ms）                                          │
├─────────────────────────────────────────────────────────┤
│ 状态管理器 ──→ 电池电量 │ WiFi信号 │ 运动状态            │
│ 表情渲染器 ──→ 待机表情 │ 说话表情 │ 充电表情            │
│ 菜单系统 ──→ 功能选择 │ 参数设置 │ 信息展示             │

└─────────────────────────────────────────────────────────┘
```

- **嵌入式屏（≤3.5"）**：LVGL（C语言），通过 SDL 在 PC 上模拟开发
- **HDMI屏（≥5"）**：PyQt / QML，开发效率高

## 五、双主控通信协议

Raspberry Pi 5 ↔ ESP32-S3 通过 UART 通信：

| 指令 | 功能 | 数据格式 |
|------|------|----------|
| `MOVE` | 移动控制 | `MOVE,<left_speed>,<right_speed>` |
| `STOP` | 紧急停止 | `STOP` |
| `IMU` | 获取姿态 | `IMU` |
| `SONAR` | 获取超声波 | `SONAR` |

## 六、关键设计决策

### 6.1 双主控架构
- **Raspberry Pi 5**：运行完整 Linux + ROS2，负责 AI 推理、屏幕 UI、导航规划
- **ESP32-S3**：实时控制电机、IMU、舵机，保证控制实时性

### 6.2 仿真先行原则
所有控制算法先在 Webots/Gazebo 中验证，再部署到实体硬件。

### 6.3 模块化设计
硬件层、驱动层、应用层严格分离，便于调试和更换硬件。
